Feed Ranking / Personalized Recommendation Systems

  • Business Metrics
    • Maximize: Watch Time, Completion Rate, Engagement (likes, shares, saves, follows, profile views)
    • Minimize: Safeguards (churn rate, drop in app opens, shorter session duration), Disengagement Signals (not interested, block, report, high scroll velocity)
  • Data and Labels
    • Outputs: Probability of watch time > k seconds, Probability of completion, Probability of engagement
    • Inputs: User Profile (demographics, app usage history), User-Content Interaction, Content Attributes, Graph Features, Contextual Information
    • Scale and Latency: Hundreds of millions of daily active users, Millions of new content items daily, Target latency ‚â§ 100 ms per request
  • System Architecture
    • Hybrid Batch + Real-Time: Batch Layer (compute embeddings daily), Real-Time Layer (session-enriched embeddings)
    • System Flow: Candidate Generation (embedding-based retrieval, ANN search), Ranking (deep personalized models), Post-Processing (re-ranking, safety filtering)
  • Data & Feature Engineering
    • Feature Types: User (demographics, usage history), Content/Creator (metadata, tags, embeddings), Interaction (likes, shares, favorites), Graph (shared followers, PageRank)
    • Feature Engineering: Cross features, Temporal features, Sequence modeling, Graph-based features
  • Model Architecture
    • Two-Tower Architecture: User Tower (encodes user features), Content Tower (encodes content features), Interaction (dot product or learned similarity)
    • Deep Learning Components: Embedding layers, Dense layers, Attention mechanisms, Multi-task learning
  • Training Strategy
    • Loss Functions: Binary cross-entropy for engagement, Regression loss for watch time, Multi-task loss
    • Training Data: Positive samples (user interactions), Negative sampling (random vs hard negatives), Temporal validation
  • Evaluation and Metrics
    • Offline Metrics: AUC, Precision@K, Recall@K, NDCG, Hit Rate, Coverage
    • Online Metrics: Click-through rate (CTR), Watch time per session, User engagement rate, Retention metrics
  • Bias Mitigation
    • Types of Bias: Selection bias (only observing interactions), Position bias (items higher get more clicks), Popularity bias, Demographic bias
    • Mitigation Strategies: Inverse propensity weighting, Position-aware models, Diversity constraints, Fairness-aware training objectives
  • A/B Testing Framework
    • Test Design: Random assignment, Statistical significance testing, Multiple testing correction, Long-term vs short-term effects
    • Key Metrics: Primary (engagement metrics), Secondary (user satisfaction, retention), Guardrail (safety, fairness metrics)

Ads Recommendation Systems

  • Business Objectives: Revenue maximization (CPM, CPC, CPA), User experience preservation, Advertiser satisfaction, Platform growth
  • System Components: Ad selection and ranking, Bid optimization, Budget pacing, Quality scoring
  • Technical Challenges: Real-time bidding, Multi-objective optimization, Ad relevance vs revenue trade-offs, Fraud detection

System Design Considerations

  • Scalability: Microservices architecture, Horizontal scaling, Caching strategies, Database sharding
  • Reliability: Fault tolerance, Circuit breakers, Graceful degradation, Monitoring and alerting
  • Data Pipeline: Real-time streaming (Kafka, Flink), Batch processing (Spark, Hadoop), Feature stores, Model serving infrastructure
