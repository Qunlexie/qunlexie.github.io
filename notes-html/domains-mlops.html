Databases

  • Structured relational:
    • e.g. MYSQL, PostgresSQL, MariaDB, SQLLite [10]
  • Unstructured/Key value store:
    • e.g. Redis, Memcache, etcd
  • Document databases:
    • e.g. MongoDB, RethinkDB
    • Not as arbitrary as key store where there is complete lack of structure
    • document databases store data in structured formats called documents
    • They often using formats like JSON, BSON, or XML
  • Graph Databases:
    • e.g. Neo4j, Dgraph
    • Has a social relationship structure: diodes, edges, properties
  • Column family databases
    • e.g Cassandra, Hbase
    • Has flexible columns to bridge the gap between relational and document databases
    • column-family databases have structures called column families.
    • Column families contain rows of data, each of which define their own format.
    • A row is composed of a unique row identifier ‚Äî used to locate the row ‚Äî followed by sets of column names and values.
  • Time series database
    • e.g Prometheus, OpenTSDB, influxDB, timescale DB
    • Data stores that focus on collecting and managing values that change over time.
    • Time series databases are heavily write oriented.
    • They are designed to handle a constant influx of incoming data [10]
  • New SQL
    • e.g MemSQL, VoltDB, Spanner, Calvin, CockroachDB, FaunaDB, yugabyteDB, PlanetScale
    • NewSQL databases follow the relational structure and semantics, but are built using more modern, scalable designs.
    • The goal is to offer greater scalability than relational databases and greater consistency guarantees than NoSQL alternatives.
    • They achieve this by sacrificing certain amounts of availability in the event of a networking partition. The trade offs between consistency and availability is a fundamental problem of distributed databases described by the CAP theorem.
    • Cap theorem asserts that in the event of a network partition, a distributed database can choose either to remain available or remain consistent, but it cannot do both
  • Multi-model databases:
    • e.g ArangoDB, OrientDB, CouchBase
    • Combining the characteristics of more than one type of database
    • multi-model databases may allow users to access and manipulate data stored in different database types within a single query
    • Multi-model databases also help maintain data consistency, which can be a problem when performing operations that modify data in many systems at once.

Cloud Computing & Cloud Storage

  • AWS
    • Amazon Elastic Cloud Compute (EC2) and Amazon Simple Storage Service (S3) are fundamentally different types of services [12].
    • EC2 allows you to run servers in the cloud with minimal effort on your part
      • Most Amazon EC2 server instances are virtual machines hosted on Amazon Web Services infrastructure
    • S3 while the other is designed for storing large amounts of static data and ideal for data backups (storage).
      • To get started with S3, users need to create a "bucket" - the endpoint resource where all uploaded data will virtually exist
  • Azure & GCP
    • similar and have cloud storage, compute services like AWS

Batch Processing Pipeline

  • Apache Airflow (Batch Pipeline)
    • Airflow is a processing platform for developing, scheduling, and monitoring batch-oriented workflows [1].
    • Very similar to dataswarm where you can stick different operators together to run a pipeline task
    • Can't be used for continuous infinite processing of data (only batch)
  • Google Kubeflow (Batch Pipeline)
    • Does the same thing as Airflow

Big Data Processing & Analytics

  • Apache Kafka (Stream Processing/ real time transport)
    • Realtime stream processing, storage and analytics system for publishing, subscribing, storing, and processing streams of records in real time.
    • Apache calls these Topics which are used to store real time events/logs [se video in 3]
  • Apache Spark (Unified Data Processing & Analytics)
    • Unified analytics engine for big data and machine learning [8]
    • Can connect to multiple databases and do data analytics, and ML (MLlib)
    • Easy to use and can be with SQL (sparksql) or python (Pyspark)
    • Spark can be 100x faster than Hadoop for large scale data processing by exploiting in memory computing and other optimizations
    • Connects with multiple data sources e.g. Hadoop, MySQL, psotgresSQL, MongoDB, Kafka, Cassandra, elastic,
  • Amazon kinesis
    • same as Apache Kafka (stream processing or real time transport)
  • Hadoop (Big Data Processing & storage)
    • Apache Hadoop is an open source software framework that provides highly reliable distributed processing of large data sets [9]
    • Hadoop, known for its scalability, is built on clusters of commodity computers, providing a cost-effective solution for storing and processing massive amounts (MapReduce for parallel processing) of structured, semi-structured and unstructured data with no format requirements (HDFS for storing).

Containerization

  • Docker (Container)
    • A Docker is a software tool that helps in building, deploying our applications efficiently. Docker is used to provide isolated environments enclosing the application and its dependencies [2]
  • Kubernetes (Container Management)
    • Kubernetes is an open-source software tool that can be used to manage containerized applications without manual intervention. In short Kubernetes can be used to manage several docker containers [2].

Data Platformization

  • Snowflake
    • Snowflake is a cloud data platform built on the foundation of a data warehouse
    • Enables customers to consolidate data into a single source of truth (since data come from various databases)
    • Analytics can be done in snowflake to drive meaningful business insights, build data-driven applications, and share data.
    • Multi-cloud platform that can be used with different cloud providers Azure, AWS, GCP [4]
  • Databricks
    • Databricks is similarly a cloud data platform but built on the foundation of a data lake (either structured or unstructured data).
    • That data lake is used for data storage but its purpose is focused on enabling data scientists to leverage machine learning applications to analyze the data.
    • The platform is primarily geared towards data science and machine learning applications [4]
  • AWS Sagemaker
    • Deploying and monitoring ML models [11]
    • Uses input from AWS cloud
    • Supports most ML and DL models
    • Simplifies the MLops work including monitoring, yuperparameter tuning, retraining etc.
