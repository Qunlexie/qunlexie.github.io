Detection

  • TLDR: object detection predicts tow things:
    • the bounding box coordinates
    • and class label
  • Starts with the input image (do some preprocessing)
  • Backbone feature extraction network using a CNN or ConvNET e.g. AlexNet, Microsoft's ResNET (remember ResNET solved the problem of vanishing/exploding gradient using skip connections and that's why it sis still popular till date [5]) used of extracting useful features
  • This features are then passed through a Region Proposal network RPN (this network is specific Faster Regional Convolutional Network - Facebook's FasterRCNN) which builds anchor or Region of Interest, ROI boxes around the objects of interest [6]. Next it does two type of detection:
    • Objectiveness: if an object is there (binary classification)
    • Bounding Box prediction: If object is there predict the x-y coordinates using linear regression
  • Final step is the object classification either car, goat, man
  • Popular Detection Models: FasterRCNN, Mask RCN, Cascade RCNN (2 stage methods optimized for detection accuracy), YOLO, SSD, RetinaNET (1-stage methods optimized for inference speed)
  • Popular Detection Datasets: Microsoft's Coco dataset

Mask Segmentation

  • A special case of detection is called Mask Segmentation
  • The ROI boxes are first layered with the ground truth to obtain an Intersection Over Union (IoU)
  • IoU > 0.5 is selected as a true region of interest
  • After that the segmentation mask is added to the architecture to obtain the mask segmented output [6].
  • Popular Mask Segmentation Models: FasterRCNN, Mask RCNN [6]
  • Popular Mask Segmentation Datasets: Microsoft's Coco dataset

Classification

  • Popular Classification Models: ResNET, VGG, AlexNet, DenseNet, EfficientNet
  • Popular Classification Datasets: ImageNet, CIFAR-10, CIFAR-100

Key Concepts

  • Data Augmentation: Techniques like rotation, flipping, cropping, color jittering to increase dataset diversity
  • Transfer Learning: Using pre-trained models (e.g., ImageNet) and fine-tuning for specific tasks
  • Feature Maps: Intermediate representations learned by CNN layers
  • Convolution Operations: Sliding window operations to detect features like edges, textures, patterns
  • Pooling: Downsampling operations (max pooling, average pooling) to reduce spatial dimensions
  • Activation Functions: ReLU, Leaky ReLU, Swish for introducing non-linearity
  • Batch Normalization: Normalizing inputs to each layer for stable training
  • Dropout: Regularization technique to prevent overfitting
